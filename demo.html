Architecture Deep Dive

HA WordPress: The SA Conversation

Internet → Route 53 → CloudFront (WAF: OWASP/SQLi rules) ↓ ↓ ALB (HTTPS) S3 (Media, OAC-protected) ↓ ┌────────────────────────────────────────┐ │ AZ-A EC2 ASG (IMDSv2, SSM) │ │ AZ-B EC2 ASG (IMDSv2, SSM) │ ← Auto Scaling: CPU + ALB req/target │ AZ-C EC2 ASG (IMDSv2, SSM) │ └────────────────────────────────────────┘ ↓ RDS MySQL Multi-AZ (Primary + Standby, KMS CMK, <2 min failover) ElastiCache (Session caching — removes DB read load) CloudWatch (Alarms: p99 >500ms, 5xx >1%)
Reliability vs Cost trade-off: Multi-AZ RDS adds ~2× cost. Justified here for zero-SPOF requirement — documented in Well-Architected review.
Security design: No SSH, no long-lived keys. IMDSv2 enforced via Launch Template, SSM Session Manager for shell access, WAF before ALB.
