Highly Available WordPress on AWS

3-AZ, zero-SPOF architecture with EC2 Auto Scaling, RDS Multi-AZ failover (<2 min), CloudFront CDN, WAF (OWASP/SQLi), KMS CMK encryption, IMDSv2, and SSM Session Manager. Designed to survive the loss of any single Availability Zone with no manual intervention.

Multi-Account AWS Landing Zone

Enterprise governance across 4 OUs and 10+ accounts. Transit Gateway hub-and-spoke for network isolation, IAM Identity Center SSO (SAML 2.0, MFA enforced), 8 SCPs, delegated GuardDuty and Security Hub, and CloudTrail centralised to an immutable Log Archive account.

Security

SCPs are the primary guardrail — deny root API calls, block public S3 ACLs, require IMDSv2, enforce region boundaries

Reliability

Transit Gateway with non-overlapping CIDRs; network account owns all routing — single failure domain for routing changes

Operational Excellence

One identity source (IAM Identity Center); CloudTrail centralised to Log Archive with MFA-Delete S3 bucket

Scale

OU structure supports 100s of accounts; new account vending via Account Factory takes <15 min

Serverless Task Management API

Full CRUD REST API at 90% lower cost than EC2+RDS. Five Lambda functions on Graviton2/arm64, API Gateway HTTP API, DynamoDB PAY_PER_REQUEST with GSI, AWS X-Ray distributed tracing, and GitHub Actions OIDC CI/CD — eliminating all long-lived secrets.

Cloud Security & Compliance Framework

Defence-in-depth across five layers: perimeter (WAF, Security Groups, NACLs), identity (IAM least-privilege, IMDSv2, Secrets Manager), data protection (KMS CMK, TLS 1.2+), detection (GuardDuty, Security Hub, Config, Inspector), and automated response (EventBridge → Lambda isolation + SNS alerts).

IAM GuardDuty Security Hub Compliance

Perimeter

WAF OWASP rules at edge; CloudFront enforces HTTPS; Security Groups use least-privilege; NACLs as stateless backstop

Identity

Permission Boundaries + SCPs prevent privilege escalation; IMDSv2 blocks SSRF; Secrets Manager rotates creds every 30 days

Detection

GuardDuty ML + Config >150 rules + Inspector CVE scanning all feed Security Hub (CIS benchmark score)

Response (<60 sec)

Security Hub → EventBridge → Lambda auto-isolates compromised EC2 (quarantine SG); SNS alerts on-call within seconds
